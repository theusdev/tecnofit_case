Regras de Negócio

1. Recorde Pessoal

O sistema agrupa todos os recordes de um usuário e seleciona o maior valor registrado.

Exemplo: Se Joao registrou 150kg, 160kg e 180kg no Deadlift, seu recorde pessoal é 180kg.

2. Ordenação Decrescente

O ranking é ordenado do maior para o menor valor de recorde pessoal.

Exemplo: 190kg → 180kg → 170kg

3. Empates Compartilham Posição

Quando dois ou mais usuários possuem o mesmo recorde pessoal, eles compartilham a mesma posição no ranking.

Exemplo: Jose (180kg) e Paulo (180kg) ficam ambos na 2ª posição.

4. Próxima Posição Após Empate (Regra RANK)

Após um empate, a próxima posição considera todos os competidores anteriores.

Exemplo: Se há 1 pessoa em 1º e 2 pessoas em 2º (empate), a próxima posição é a 4ª, não a 3ª.

PosiçãoUsuárioRecorde
1ºJoão190 kg
2ºJosé180 kg
2ºPaulo180 kg
4ºMaria170 kg

Note que não existe a 3ª posição devido ao empate.

5. Data do Recorde

A data exibida corresponde ao momento em que o maior valor foi registrado.

Se o recorde de 180kg foi atingido em duas datas diferentes, exibe a mais recente.

6. Window Function (RANK)

A API utiliza a função RANK() do MySQL 8 para calcular as posições corretamente.

RANK() OVER (ORDER BY pr.value DESC) - Garante empates com posições corretas
